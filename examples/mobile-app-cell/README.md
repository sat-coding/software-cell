# Mobile App Cell Example

This example shows how a software cell could be configured for a mobile product.

## Suggested role mapping

- Coordinator -> `product/product-manager.md`
- Engineer -> `engineering/engineering-mobile-app-builder.md`
- Verifier -> `testing/testing-reality-checker.md`
- Designer -> `design/design-ui-designer.md`
- Runtime -> `engineering/engineering-devops-automator.md`

## Core rule

The team does not stop at implementation.
It must be able to:
- run the app
- inspect the result on emulator or device
- collect screenshots/logs
- document what happened

## Example cycle

1. Build onboarding screen
2. Run app on emulator
3. Capture screenshot
4. Verify CTA/navigation
5. Review layout visually
6. Patch issues
7. Record result in `QA_LOG.md`
